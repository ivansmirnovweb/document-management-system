# 106 — Replace raw ID display with human-readable entity labels across UI, search, and exports

## Goal

Убрать из пользовательского интерфейса и экспортов отображение «сырых» ID сущностей и заменить их на понятные человеку значения (ФИО, логин, наименование контрагента, регистрационный номер документа).

## Context

Сейчас в ряде мест вместо читаемых данных выводятся числовые идентификаторы (`ownerId`, `executorId`, `employerId`, `outSenderEmployerId`, `lastChangedById` и т.п.). Это ухудшает UX и мешает анализу записей.

## Scope

- Провести аудит frontend-экранов, где рендерятся документы/детали/поиск/архив/удаленные записи/отчеты.
- Заменить отображение ID на человекочитаемые поля:
  - пользователь: `displayName` (fallback: `username`);
  - контрагент: `shortName` (fallback: `fullName`);
  - документ: `registrationNumber` или `title` по контексту.
- Обновить API-ответы/DTO только там, где сейчас не хватает данных для отображения имен (без изменения бизнес-логики).
- Обновить экспортные представления, чтобы в колонках не было «голых» ID.

## Requirements

- В UI не должно быть мест, где конечный пользователь видит технический ID вместо читаемого значения, кроме специальных админ-диагностических разделов (если они явно помечены как технические).
- Для связей с `null` должны быть явные подписи: `—` или `Не указан`.
- Должны быть fallback-значения, чтобы интерфейс не ломался при неполных данных.
- Не нарушить текущие права доступа и существующие API-контракты без необходимости.

## Constraints

- Не переносить бизнес-логику в frontend.
- Не делать широкий рефакторинг вне задачи отображения сущностей.
- Не добавлять новые зависимости без необходимости.
- Backend остается source of truth.

## Expected files

- `apps/web/src/features/**`
- `apps/web/src/entities/**`
- `apps/web/src/shared/**`
- `apps/api/src/documents/**` (точечно, если нужны дополнительные поля в ответах)
- `shared/**` (только если потребуется корректировка общих контрактов)

## Definition of done

- Основные пользовательские таблицы/карточки/детали/экспорты показывают имена/названия вместо ID.
- Поиск и фильтры отображают те же человекочитаемые значения.
- Фолбэки для `null`/отсутствующих связей работают корректно.
- Изменения проверены вручную на активных, архивных и удаленных документах.
