# TZ Notes (рабочие уточнения)

## Правила использования

1. `docs/TZ.md` — первоисточник требований.
2. Если формулировка в задаче (AGENTS/tasks/*) краткая, детализацию брать из `TZ.md`.
3. Если в `TZ.md` не хватает конкретики для поля/API, фиксировать интерпретацию здесь перед реализацией.

## Текущее уточнение по задачам 083+

- 083: классификация документов `incoming/outgoing/internal` реализуется явно и проверяется на уровне БД + API + UI.
- 084: учет резолюций реализуется отдельной сущностью, связанной с документом, с обязательными полями из формулировки задачи (`author`, `text`, `date`, `dueDate`, `linkedDocument`).
